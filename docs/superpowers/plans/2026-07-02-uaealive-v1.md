# UAE ALIVE v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Execution in this project runs via multi-agent Workflows (ultracode); every implementing agent MUST be given: this plan, the spec (`docs/superpowers/specs/2026-07-02-uaealive-v1-design.md`), and the Global Constraints below.

**Goal:** Browser-based AI + WebAR heritage platform for Al Fahidi Historical District (Dubai) — cinematic Arabic-first demo frontend over a production-grade FastAPI/PostGIS backend.

**Architecture:** Monorepo — `apps/web` (Next.js 15 App Router + R3F + MapLibre) talks only to `apps/api` (FastAPI + SQLAlchemy 2 + PostGIS) under `/api/v1` with a uniform `{ok,data,error}` envelope. All AI calls go server-side through the user's LiteLLM proxy with an automatic canned-answer fallback. Seed knowledge base lives in `data/*.json` + committed OSM-derived GeoJSON.

**Tech Stack:** Next.js 15/TypeScript strict/Tailwind/React Three Fiber + drei/MapLibre GL/framer-motion/next-intl · FastAPI/SQLAlchemy 2/GeoAlchemy2/Alembic/Pydantic v2/slowapi/openai-sdk (pointed at LiteLLM) · postgis/postgis:16-3.4 · Docker Compose + nginx.

## Global Constraints

- API base path: `/api/v1`. Every response uses envelope `{"ok": bool, "data": ..., "error": str|null}` — except SSE streams and `/api/v1/geo/district` (raw GeoJSON FeatureCollection).
- Env vars (`.env`, mirrored in `.env.example` with placeholders): `DATABASE_URL`, `LITELLM_BASE_URL`, `LITELLM_API_KEY`, `AI_MODEL_CHAT=claude-account-haiku-4-5`, `AI_MODEL_COPILOT=claude-account-sonnet-4-6`, `ADMIN_PASSWORD`, `CORS_ORIGINS`, `NEXT_PUBLIC_API_URL`. **The real key is NEVER committed and NEVER sent to the browser.**
- Locales: `ar` (default, RTL) and `en`. Every user-facing string lives in per-namespace files `apps/web/src/messages/{ar,en}/{common,landing,map,twin,stories,chat,arx,hunt,community,admin}.json`, merged in `i18n/request.ts` — one namespace per page so parallel agents never touch the same file. All seed content is bilingual `{"ar": "...", "en": "..."}`.
- **Implementing agents do NOT run git commands** — the orchestrator commits between phases (avoids index lock races between parallel agents).
- Python 3.12, `ruff` clean; TypeScript `strict: true`, `next build` must pass with zero type errors.
- Design tokens ONLY from `apps/web/src/lib/theme.ts` + Tailwind config (§Task 8). No hardcoded hex in components.
- Files ≤ ~400 lines; split by responsibility. Functions < 50 lines.
- Commit after every task (conventional commits, no attribution footer — user has attribution disabled).
- Rate limits (slowapi): chat/copilot `20/minute/IP`, submissions `5/minute/IP`, analytics `60/minute/IP`.
- Anonymous identity: client-generated `device_id` (UUID v4, localStorage) — no accounts in v1.

## Interface Contracts (single source of truth)

**Envelope (Python):**
```python
def envelope(data=None, error: str | None = None) -> dict:
    return {"ok": error is None, "data": data, "error": error}
```

**Public endpoints:**

| Method+Path | Returns (in `data`) |
|---|---|
| GET `/api/v1/pois?kind=&lang=` | `PoiOut[]` |
| GET `/api/v1/pois/{slug}` | `PoiDetailOut` (POI + its stories) |
| GET `/api/v1/stories?poi=&audience=` | `StoryOut[]` |
| GET `/api/v1/timeline` | `PeriodOut[]` |
| GET `/api/v1/characters` | `CharacterOut[]` (NO persona_prompt leaked) |
| GET `/api/v1/events?upcoming=true` | `EventOut[]` |
| GET `/api/v1/geo/district` | raw GeoJSON FeatureCollection |
| POST `/api/v1/chat/{character_slug}` | SSE stream (below) |
| POST `/api/v1/copilot/tour` | SSE stream (below) |
| POST `/api/v1/community/submissions` | `{id, status: "pending"}` |
| POST `/api/v1/hunt/checkin` | `{correct: bool, stop?, badge?, progress: {found, total}}` |
| GET `/api/v1/hunt/stops` | `HuntStopOut[]` (without secret codes) |
| GET `/api/v1/hunt/progress?device_id=` | `{found: string[], badges: BadgeOut[]}` |
| POST `/api/v1/analytics/track` | `{tracked: true}` |

**Admin endpoints** (header `Authorization: Bearer <ADMIN_PASSWORD>`, 401 otherwise): `POST/PUT/DELETE /api/v1/admin/pois[/{id}]`, same for `stories`, `events`; `GET /api/v1/admin/submissions?status=pending`, `POST /api/v1/admin/submissions/{id}/approve|reject`; `GET /api/v1/admin/analytics/summary` → `{total_events, by_page: [...], top_pois: [...], by_lang: [...], chat_messages}`.

**SSE wire format (chat + copilot):** `Content-Type: text/event-stream`; each chunk `data: {"delta": "text"}\n\n`; terminal event `data: {"done": true, "source": "live"|"fallback"}\n\n`. Request bodies — chat: `{"messages": [{"role": "user"|"assistant", "content": str}], "locale": "ar"|"en"}` (max 20 messages, each ≤ 2000 chars); copilot: `{"interests": str[], "duration_min": int, "audience": str, "locale": "ar"|"en"}`.

**PoiOut:** `{id, slug, kind, name: {ar,en}, summary: {ar,en}, lat, lng, era_built, accessibility: {wheelchair: bool, audio: bool}, hero_image, order}`. **StoryOut:** `{id, poi_slug, audience: "tourist"|"kids"|"expert", title: {ar,en}, body: {ar,en}, sources: str[]}`. **CharacterOut:** `{slug, name: {ar,en}, role: {ar,en}, greeting: {ar,en}, avatar}`. **PeriodOut:** `{key: "1950"|"1970"|"1990"|"today", name: {ar,en}, description: {ar,en}, palette: {sky, ambient, accent}}`. **EventOut:** `{id, slug, kind, title: {ar,en}, description: {ar,en}, starts_on, ends_on, location: {ar,en}}`.

**GeoJSON feature properties:** buildings `{kind: "building", height: number (m, default 5), barjeel: bool, poi_slug?: string}`; paths `{kind: "alley"|"path"}`; district boundary `{kind: "boundary"}`.

**Frontend API client (`apps/web/src/lib/api.ts`):** typed functions `getPois()`, `getPoi(slug)`, `getStories(params?)`, `getTimeline()`, `getCharacters()`, `getEvents()`, `getDistrictGeo()`, `checkin(deviceId, code)`, `getHuntStops()`, `getHuntProgress(deviceId)`, `submitContribution(payload)`, `track(event, meta?)`, plus `streamChat(slug, messages, locale, onDelta, onDone)` and `streamTour(body, onDelta, onDone)` using `fetch` + ReadableStream SSE parsing. All types in `apps/web/src/lib/types.ts` mirroring the schemas above.
